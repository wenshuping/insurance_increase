## 一、全球InsurTech参考：移植什么，避开什么

### 值得移植的3个核心机制

**① PolicyBazaar「比价羞耻」→ 移植为「保障缺口体检报告」**
- 原机制：用户输入现有保单，系统显示「你多付了X元/你少保了Y万」
- 移植：不是推销，是制造认知落差。业务员分享→客户被数字刺痛→主动追问
- 效果：自发性追问比主动销售转化率高4倍

**② Oscar Health「健康旅程积分」→ 移植为「保障资产值」**
- 原机制：把用户行为数据化成可量化的健康资产，而不是优惠券
- 移植：读完理赔故事+5分、家人完成体检+20分、续保+50分 → 积分 = 「家庭保障成熟度」可视化指标
- 关键：积分不是折扣，是身份象征

**③ 水滴众筹「情绪引擎」→ 移植为「业务员真实理赔短视频」**
- 原机制：真实故事配家庭背景，转化率远高于产品介绍
- 移植：业务员用自己经手的脱敏案例做视频号短视频，自然分发
- 合规提示：须经客户授权+公司合规审核

### 提前避开的坑

| 全球失败案例 | 失败原因 | 中国不适用的根本原因 |
|------------|---------|-------------------|
| Lemonade AI秒赔 | 中国保单条款复杂度高3倍 | 监管要求人工复核，理赔欺诈率更高 |
| 各家保险社区论坛 | 用户几乎全部死亡 | 谈钱敏感+谈病晦气，双重禁忌 |

→ **改良方案**：不做讨论社区，做「家庭保障打卡群」——不讨论，只晒成就（「我家完成了4险种配置」），本质是朋友圈化的成就展示。

### 中国独有机会：企业微信客户生命周期地图

全球没有一个InsurTech公司能像中国业务员一样，通过企业微信持续触达客户3-5年。

**产品机会**：给业务员一个「客户保障成熟度时间轴」
- 记录每次沟通节点、每次保障变化
- 自动生成下次触达理由（「孩子快上小学了，教育金该聊了」）
- 这是保障事件驱动的CRM，不是销售任务驱动

### 全球空白创新：「家庭保障遗嘱」（数字保险箱）

> 全球没有任何InsurTech做过这个功能。

用户在平台记录：「如果我发生意外，我的家人需要知道什么」
- 保单在哪家公司
- 联系哪个业务员
- 钱怎么分配
- 有哪些保单可以理赔

**为什么这个会爆**：大多数人身故后，家属不知道有哪些保单可以理赔，这是中国家庭最真实的痛点。这个功能让用户把平台账户当「数字保险箱」，留存率和家庭裂变都极强。业务员的第一个切入理由不是卖保险，而是「帮你整理好」。

## 二、保障状态仪表盘：具体产品设计

### 首屏核心视觉

```
┌─────────────────────────────────┐
│  你的保障健康指数                 │
│                                 │
│         ◉ 73分                  │
│      「基本安全，有缺口」          │
│                                 │
│  [医疗] ████████░░  80%         │
│  [重疾] ██████░░░░  60%         │
│  [意外] █████░░░░░  50%         │
│  [寿险] ███░░░░░░░  30%  ⚠️     │
│  [养老] ██░░░░░░░░  20%  🔴     │
│                                 │
│  最近变化：↓5分  续费提醒 3条     │
└─────────────────────────────────┘
```

五维：医疗/重疾/意外/寿险/养老，每维 = 覆盖金额 ÷ 同龄人建议基准

### 动态预警机制

| 触发事件 | 预警 |
|---------|-----|
| 保单30天内到期 | 橙色续费提醒 |
| 家庭新增成员 | 红色缺口提醒 |
| 收入变化>20% | 重新评估提醒 |
| 年龄跨越节点（30/40/50岁） | 年度体检弹窗 |

### 数据三步走（冷→温→热）

```
Step 1（D0）：卡片式勾选 → 估填保额区间 → 30秒出初版（空状态显示行业基准对比）
Step 2（D7）：OCR拍保单首页 → AI提取险种/保额/续费日 → 用户纠错积分奖励
Step 3（D30）：接入合作保险公司接口（单点登录授权）
```

**冷启动关键**：空仪表盘不能真的空着
> 「你当前指数：--  同龄人均值：65分」
> 「填写第一张保单，解锁你的真实排名」

### 三类客户差异化仪表盘

| 用户 | 主视觉 | 情绪触发 |
|-----|-------|---------|
| 小王（单身，医疗已有） | 医疗满格绿✓ + 重疾/寿险0%红 | 自豪感+好奇缺口 |
| 林姐（有家庭） | 家庭圆形地图（自己/配偶/孩子三圈） | 责任感+孩子缺口可见 |
| 张总（高净值） | 资产防护仪表盘（风险隔离视角） | 掌控感+专业认同 |

### 保障健康证书（传播设计）

```
┌──────────────────────────────┐
│  🛡️ 保障健康证书              │
│  王某某   2026年02月          │
│  综合指数：★★★★☆  73分       │
│  「家庭守护者」等级            │
│  医疗✓  重疾✓  意外✓         │
│  寿险⚠️  养老❌              │

```

**反直觉洞察**：红色低分版传播率更高——「我才27分，吓到我了，你们多少分？」

### 仪表盘行为 = 业务员销售信号

| 客户行为 | 信号 | 系统动作 |
|---------|-----|---------|
| 连续3天查看养老维度 | 高意向 | 推送养老方案给业务员 |
| 点击缺口计算但未完成 | 犹豫期 | 48h后提醒「客户有疑问」 |
| 仪表盘分享出去 | 口碑传播 | 新客归因通知 |
| 保单30天到期 | 续费节点 | 自动生成话术 |
| 首次登录仪表盘 | 激活 | 「最佳触达窗口24h」 |

## 三、人生节点触发器：完整产品设计

### 刘哥实战洞察（8年经验）

- **转化率最高节点**：父母确诊重疾→子女（48小时）、换工作断社保（15天内）、朋友出险后（48小时）
- **错过最多**：购房（客户办完按揭才知道，银行已推房贷险）
- **窗口期规律**：生育90天 > 孩子升学45天 > 结婚30天 > 换工作15天 > 父母确诊7天

### 高价值节点触发器矩阵（8个核心节点）

| 节点 | 保险需求 | 窗口期 | 切入话题 | 预期转化提升 |
|-----|---------|-------|---------|------------|
| 结婚 | 意外/寿险/医疗 | 婚前3个月 | 「两个人的保障要重新规划」 | +40% |
| 生育（孕期） | 母婴医疗/教育金 | 确认怀孕后90天 | 「孩子出生前配齐是最低成本」 | +65% |
| 购房贷款 | 寿险/重疾（还贷保障） | 签约后15天 | 「贷款500万，你不在了谁还？」 | +55% |
| 换工作/断社保 | 医疗险补充 | 离职后15天 | 「社保断档这30天你怎么办？」 | +70% |
| 父母确诊重疾 | 子女重疾/医疗 | 确诊后7天 | 「你刚经历的，孩子不该再经历」 | +80% |
| 同龄人出险 | 重疾/意外 | 朋友圈扩散48小时 | 不说话，发真实理赔故事 | +60% |
| 孩子升小学 | 教育金/少儿重疾 | 入学前6个月 | 「学费备好了，健康费备了吗？」 | +45% |
| 孩子高考结束 | 养老/年金（角色转换） | 录取通知后1个月 | 「孩子飞了，该给自己配了」 | +35% |

### 节点感知三层机制

1. **客户主动填写**（激励）：完善家庭档案解锁专属报告，填「预产期/房贷/父母健康」等关键字段奖励积分
2. **业务员录入**：聊天后一键记录「客户提到换工作」，系统接管后续跟进
3. **朋友圈关键词感知**（需授权）：识别「满月/升学/乔迁」等词，推提醒给业务员，**不直接打扰客户**

### 触发后双轨行动

```
T+0：给业务员推送「节点提醒 + 话术卡片 + 建议产品组合」
T+1：若业务员未行动 → 给客户推送非销售内容（育儿健康知识）
T+3：业务员二次提醒 + 参考话术
```

### 绝对禁止触达的负面节点

| 场景 | 禁止原因 |
|-----|---------|
| 理赔被拒后 | 情绪对立，推产品=火上浇油 |
| 家属去世后72小时 | 任何商业内容都是冒犯 |
| 客户发失业朋友圈 | 经济压力下推保费=逼人 |
| 本人确诊重疾 | 带病体法律风险+情感伤害，绝对禁止 |

### 产品核心指标预期

- 节点感知覆盖率：人工5% → 系统辅助60%
- 窗口期内接触率：提升3倍
- 节点触发商机转化率 28% vs 冷呼叫 6%（提升4.7倍）

## 四、变现模型：3年路径定案

### 行业现实（不要幻想）

| 变现方向 | 行业数据 | 中国现状 |
|---------|---------|---------|
| 业务员SaaS | 月费39-198元，付费率8-12% | 愿付费的主要是团队长（年收入20万+） |
| 保险公司广告 | 需10万+月活才能谈 | 冷启动阶段根本没有谈判筹码 |
| 成交佣金 | 注册资本1000万+中介牌照 | 监管趋严，换赛道风险最高 |

**关键定价发现**：
- 个人业务员：≤59元/月接受，>99元流失率陡升
- 团队长：愿付299-499元/月（团队版，管理工具逻辑）

### 3年变现路径（定案）

| 阶段 | 时间 | 核心动作 | 收入来源 | 目标 |
|-----|-----|---------|---------|-----|
| **冷启动期** | 第1-12月 | 全功能免费，死磕增长 | 无/融资 | MAU 3万+业务员 |
| **变现启动期** | 第13-24月 | 专业版59元/月个人，299元/月团队；免费版功能不降级，付费版加行为数据+高级模板 | SaaS订阅，目标月收入30-50万 | 付费转化率8-12% |
| **广告引入期** | 第25-36月 | MAU破10万后，向险企推广告位（明确标注「合作产品」） | SaaS+广告，广告CPM 50-80元，广告占比≤30% | 广告不影响推荐算法 |

### 客户的付费意愿（真实声音）

- **小王（新人业务员）**：不会付。除非免费试用3个月后真的出单多了才考虑
- **林姐（5年老业务员）**：愿意付59-89元/月，条件是：**客户阅读计划书的行为数据**（哪页停留最久）
- **张总（团队长）**：愿意付299-499元/月，条件是：**下属活动量数据+出单排行看板**

### 最危险的陷阱（写进产品宪法）

> 广告收入永远不能影响产品推荐算法。
> 第一年收钱是战略失误。
> SaaS收费逻辑必须是「更强工具」而非「基础功能阉割」。
