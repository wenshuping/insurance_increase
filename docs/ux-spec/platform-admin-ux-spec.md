# 保险裂变增长平台 — 平台运营端 UI/UX 设计规范

**版本：** v1.0.0
**日期：** 2026-02-19
**技术栈：** Vue3 + Element Plus + Tailwind CSS（PC端后台）
**目标用户：** 平台运营人员 / 保险公司管理员

---

## 一、设计系统基础

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| 信息密度优先 | ToB后台优先展示信息密度，不牺牲功能换取留白 |
| 操作可回溯 | 危险操作必须有确认弹窗 + 操作日志 |
| 状态可感知 | 页面所有异步状态（加载/空状态/错误）都要有明确提示 |
| 权限最小化 | 界面仅渲染当前角色有权限的元素 |
| 数据可信 | 所有统计指标附带更新时间戳 |

### 1.2 色彩系统

#### 主色板

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-primary` | `#1E40AF`（蓝-800） | 主操作按钮、激活导航、链接 |
| `--color-primary-light` | `#DBEAFE`（蓝-100） | 主色背景、Tag背景 |
| `--color-primary-hover` | `#1D4ED8`（蓝-700） | 按钮hover态 |

#### 语义色

| Token | 色值 | 用途 |

| `--color-danger` | `#DC2626`（红-600） | 驳回、禁用、告警、负向指标 |
| `--color-info` | `#6B7280`（灰-500） | 次要信息、已归档 |

#### 中性色

### 1.3 字体规范

```

  中文：PingFang SC / Microsoft YaHei（系统字体降级）
  英文/数字：Inter（Google Fonts）
  等宽：JetBrains Mono（代码/ID展示）

  --text-xs:   12px / line-height: 1.5  （辅助说明、标签）
  --text-sm:   13px / line-height: 1.6  （表格单元格、次要文字）
```

## 二、通用组件规范

### 2.1 确认弹窗规范（危险操作）

```javascript
ElMessageBox.confirm(
  '此操作将禁用该业务员账号，禁用后将无法登录系统。',
  '确认禁用',
  {
    confirmButtonText: '确认禁用',
    cancelButtonText: '取消',
    type: 'warning',
    confirmButtonClass: 'el-button--danger',
  }
)
```

- 标题：动词 + 操作对象（「确认禁用」「确认驳回」「确认下架」）
- 不可逆操作标注「此操作不可撤销」
- 删除类极高风险操作需用户输入 `DELETE` 或目标名称确认

### 2.2 状态 Tag 规范

| 状态 | Tag类型 |
|------|---------|
| 正常/通过/已发布 | `type="success"` |
| 待审核/待处理 | `type="warning"` |
| 驳回/禁用/异常 | `type="danger"` |
| 已归档/已下架 | `type="info"` |

## 模块一：数据大盘（运营概览仪表盘）

**目的：** 平台运营人员和管理层获取全局经营数据，监控增长健康度，捕捉异常预警。

**页面结构：**
```
[PageHeader: 时间选择器（昨日/7日/30日/90日）+ 数据更新时间戳]
[告警横条：有未处理告警时显示，背景 warning-100]
[核心KPI卡片区：4卡片横向排列]
  业务员MAU | 客户MAU | K因子 | 整体转化率
[实时今日数据（4格）] | [增长趋势折线图（可切换维度）]
[功能使用分布环形图] | [积分流通面积图]
[告警列表：最近10条未处理告警]
```

**核心指标定义：**

| 指标 | 计算公式 | 更新频率 |
|------|---------|---------|
| 业务员MAU | 30日内登录且有操作行为的独立用户数 | 每日00:00 |
| 客户MAU | 30日内访问过H5/小程序的独立用户数 | 每日00:00 |
| K因子 | 统计周期内（邀请发出总数 × 注册转化率）/ 初始用户数 | 每日00:00 |
| 整体转化率 | 产生线索的客户数 / 完成测评的客户数 × 100% | 实时 |

**今日实时数据（5分钟刷新）：**

| 指标 | 计算方式 |
|------|---------|
| 新注册业务员 | 当日注册且完成手机验证的业务员数 |
| 新完成测评 | 当日首次完成需求测评的客户数 |
| 新生成计划书 | 当日生成的计划书总份数 |
| 新获线索 | 当日客户主动提交联系方式的次数（实时） |

**图表规范：**
- 增长趋势图：ECharts多线折线图，支持框选放大（dataZoom），Tooltip显示具体数值+环比
- 功能分布图：ECharts环形饼图，中心显示总UV，图例点击可显隐
- 积分趋势：ECharts面积图，双Y轴（发放量/消耗量）

**告警列表字段：**

| 字段 | 说明 |
|------|------|
| 告警类型 | Tag：积分异常/合规风险/内容超时 |
| 告警描述 | 具体触发原因 |
| 告警时间 | 精确到分钟 |
| 严重程度 | 高/中/低 |
| 状态 | 未处理/处理中/已关闭 |
| 操作 | 查看详情（跳转对应模块） |

**权限控制：**
- 可见：超级管理员、数据分析员、内容审核员（仅内容指标）、保险公司管理员（仅本公司数据）
- 时间筛选：超级管理员、数据分析员
- 告警处理入口：超级管理员

## 模块二：业务员管理列表

**目的：** 全面管理平台注册业务员，执行账号状态管控、异常处置、关联数据查看。

**页面结构：**
```
[PageHeader] [导出数据] [批量操作▼]
[FilterBar: 团队/注册时间/活跃度/状态/手机号|ID]
[统计摘要: 总计X人 | 正常X | 禁用X | 今日新增X]
[DataTable]
[Pagination: 默认20条/页，可选10/20/50/100]
```

**核心字段：**

| 字段 | 类型 | 宽度 | 说明 |
|------|------|------|------|
| 选择框 | Checkbox | 48px | 批量操作 |
| ID | 等宽文本 | 80px | 可复制 |
| 姓名 | 文本/链接 | 100px | 点击进详情 |
| 手机号 | 文本 | 130px | 脱敏：138****8888 |
| 所属团队 | 链接 | 120px | 跳转团队详情 |
| 注册时间 | 日期时间 | 160px | YYYY-MM-DD HH:mm |
| 活跃度 | Tag | 90px | 高活跃/中活跃/低活跃/沉默 |
| 计划书数 | 数值 | 80px | 历史累计 |
| 客户数 | 数值/链接 | 80px | 点击跳转筛选客户列表 |
| 线索数 | 数值 | 80px | |
| 状态 | Tag | 80px | 正常/禁用 |
| 操作 | 按钮组 | 180px（fixed right） | 详情/禁用或启用/重置密码 |

**活跃度定义：**

| 等级 | 定义 |
|------|------|
| 高活跃 | 近7日内登录 ≥ 3次且有业务操作 |
| 中活跃 | 近30日内登录 ≥ 1次 |
| 低活跃 | 近90日内登录 ≥ 1次 |
| 沉默 | 超过90日未登录 |

**操作设计：**

| 操作 | 触发条件 | 确认方式 | 结果反馈 |
|------|---------|---------|---------|
| 查看详情 | 点击姓名/详情按钮 | 直接跳转 | 打开详情页 |
| 禁用账号 | 状态为正常时显示 | 确认弹窗（说明后果） | 成功提示+行状态变更 |
| 启用账号 | 状态为禁用时显示 | 确认弹窗 | 成功提示+行状态变更 |
| 重置密码 | 任意状态 | 确认弹窗 | 弹窗展示临时密码，可复制 |

**批量操作：** 勾选≥1条后顶部浮现工具栏，显示「已选X条 [批量禁用] [批量发送通知] [取消选择]」，批量禁用需列出所有将被禁用账号姓名确认。

**权限控制：**
- 可见：超级管理员、数据分析员（只读）
- 禁用/启用/重置密码：仅超级管理员
- 导出：超级管理员（记录导出日志）

## 模块六：理赔故事管理

**目的：** 对业务员上传的理赔故事进行合规审核，确保来源授权合法、内容脱敏充分后发布。

**页面结构（Tabs）：**
```
Tabs: [待审核(X)] [审核中] [已通过] [已驳回] [已发布] [已下架]
```

**核心字段：**

| 字段 | 类型 | 宽度 | 说明 |
|------|------|------|------|
| ID | 文本 | 80px | |
| 故事摘要 | 文本 | 200px | 前50字，点击展开全文 |
| 险种 | Tag | 100px | 寿险/重疾/医疗/意外等 |
| 理赔金额区间 | 文本 | 100px | 脱敏后区间显示 |
| 上传业务员 | 链接 | 100px | |
| 上传时间 | 日期时间 | 160px | |
| 授权证明 | 文件链接 | 80px | 查看PDF/图片附件 |
| 脱敏确认 | Tag | 90px | 已确认/待确认 |
| 等待时长 | 文本 | 80px | 超过48小时显示红色 |
| 操作 | 按钮 | 200px | 审核/认领/查看详情 |

**审核详情页（全屏弹窗）：**
```
左侧60%：内容预览区（完整故事文本，敏感词高亮标注）+ 授权证明预览
右侧40%：合规检查清单（Checkbox逐项确认）
  ☑ 姓名已脱敏（真实姓名未出现）
  ☑ 身份证号/联系方式未泄露
  ☑ 授权证明真实有效
  ☑ 内容无夸大/误导性描述
  ☑ 禁止词检查通过
  [审核意见文本框，驳回时必填]
  操作按钮：[通过审核（绿）] [驳回（红）] [要求补充信息（橙）]
```

**审核状态流转：**
```
[待审核] → 认领 → [审核中]
                      ↓
          [已通过] → [已发布] → [已下架]
          [已驳回] → 业务员修改 → [待审核]（重新进队列）
          [要求补充信息] → 业务员补充 → [待审核]
```

**操作设计：**

| 操作 | 触发条件 | 确认方式 | 结果 |
|------|---------|---------|------|
| 通过审核 | 所有合规检查项勾选 | 无弹窗，审核意见选填 | 状态→已通过，通知业务员 |
| 驳回 | 任意时刻 | 驳回原因必填 | 状态→已驳回，通知业务员 |
| 要求补充 | 任意时刻 | 补充要求必填 | 状态→要求补充，通知业务员 |
| 发布 | 状态已通过 | 确认弹窗 | 状态→已发布，内容上线 |
| 下架 | 状态已发布 | 确认弹窗（下架原因必填） | 状态→已下架，内容下线 |
| 批量审核 | 勾选多条待审核 | 弹窗列出清单，可整体通过 | 批量更新状态 |

**权限控制：**
- 可见：超级管理员、内容审核员
- 审核操作：内容审核员及以上
- 发布/下架：仅超级管理员

## 模块七：保险翻译官内容抽检

**目的：** 对AI生成的保险翻译官内容进行月度质量抽检，确保准确率达标（目标>90%）。

**页面结构：**

## 四、全局交互规范

### 4.1 表单校验规范

| 校验类型 | 触发时机 | 展示位置 |
|---------|---------|---------|
| 必填校验 | 提交时 | 字段下方红色文字 |
| 格式校验（邮箱/手机） | 失焦后 | 字段下方红色文字 |
| 长度校验 | 实时 | 字段右下角字符计数 |
| 唯一性校验（如邮箱） | 失焦后（异步）+loading指示 | 字段下方 |
| 跨字段校验 | 提交时 | 表单顶部错误汇总 |

### 4.2 消息提示规范

| 场景 | 组件 | 位置 | 持续时间 |
|------|------|------|---------|
| 操作成功 | el-message success | 顶部居中 | 2秒 |
| 操作失败 | el-message error | 顶部居中 | 3秒 |
| 预警提示 | el-message warning | 顶部居中 | 3秒 |
| 信息通知 | el-notification | 右上角 | 4秒 |
| 重要告警 | el-alert（页面内嵌） | 页面内 | 手动关闭 |

### 4.3 批量操作规范

1. 勾选时顶部工具栏浮现（fixed定位，不随滚动消失）
2. 显示「已选 X 条」
3. 批量危险操作需二次确认，列出所有受影响对象名称/ID
4. 异步批量处理时显示进度条
5. 完成后刷新列表并显示摘要（成功X条，失败X条）

### 4.4 数据刷新规范

| 场景 | 刷新方式 |
|------|---------|
| 实时数据（仪表盘今日数据） | 定时自动刷新（300秒），显示倒计时 |
| 列表页操作后 | 操作成功后自动刷新当前列表 |
| 手动刷新 | 每个有数据的页面右上角提供「刷新」图标按钮 |
| 统计数据 | 显示「数据截至 YYYY-MM-DD HH:mm」时间戳 |
