## 一、设计系统基础

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| 正向优先 | 所有文案、图标、颜色传递「成就感、安全感、被爱」，不传递恐惧 |
| 触达即传播 | 每个关键页面都有自然的分享出口，不强迫、不打断流程 |
| 30秒完成 | 核心漏斗（测评）每步不超过2个问题，加载不超过2秒 |
| 空状态有温度 | 无数据时不显示空白，展示引导叙事或同龄人均值参考 |
| 双端一致 | H5与小程序视觉一致，交互适配各自平台规范 |

### 1.2 色彩系统

**主色调（温暖蓝绿，拒绝焦虑红）**

```
主色 Primary:      #2B7FBF  信任蓝（主按钮、高亮数据、品牌标识）
辅助色 Secondary:  #3DAA85  守护绿（进度条满格、成就态、完成标识）
暖调 Warm:         #F5A623  活力橙（积分、勋章、庆典、轻度提醒）
```

**语义色彩（替代传统红绿体系）**

```
完整/成功:   #3DAA85  守护绿（替代绿色警告）
提醒/盲区:   #F5A623  暖橙（替代危险红，减少焦虑联想）
关注/升级:   #2B7FBF  信任蓝
中性信息:    #6B7A8D  灰蓝（说明文字、辅助信息）
```

> **设计红线：** 禁止使用纯红色 `#FF0000` 系列作为界面警告色。保险场景中红色触发恐惧联想，所有「预警」类信息改用暖橙 `#F5A623`。仅答题错误反馈场景可用极淡红（#FFF0F0背景）。

**背景色系**

```
页面背景:    #F6F8FB  极浅灰蓝（冷静清洁）
卡片背景:    #FFFFFF  纯白
渐变背景:    linear-gradient(160deg, #EBF4FF 0%, #E8F8F2 100%)
深色蒙层:    rgba(15, 30, 55, 0.60)
```

**文字色系**

```
主标题:      #0F1E37  深海蓝（最高对比度）
副标题:      #2C3E55  深灰蓝
正文:        #3D4F63  中灰蓝（on #FFFFFF 对比度 6.8:1，超WCAG AA标准）
说明/辅助:   #6B7A8D  灰蓝
占位符:      #A8B4C0  浅灰
```

### 1.6 动效规范

```
微交互（点击反馈）:    150ms ease-out
卡片展开:             300ms cubic-bezier(0.4, 0, 0.2, 1)
页面过渡:             350ms ease-in-out（右侧滑入）
数字滚动（分数）:      1200ms，0→目标值，countUp 效果
进度条动画:           800ms cubic-bezier(0.4, 0, 0.2, 1)
庆典动效（购买/理赔）: 2000ms 粒子/烟花效果
骨架屏 shimmer:       1400ms 循环，background-position 动画
```

> **无障碍要求：** 所有动效支持 `prefers-reduced-motion: reduce` 媒体查询，检测到系统减弱动效设置时跳过或简化。

### 1.7 图标规范

- 全部使用 SVG 图标，线条风格，2px stroke，圆头圆角
- **禁止使用 emoji 作为功能图标**（emoji仅可用于情绪表达场景的文案内）
- 推荐图标集：Lucide Icons（开源，MIT协议）
- 尺寸规范：导航图标 24x24px / 列表图标 20x20px / 大型展示图标 48x48px

### 1.8 页面布局规范

```
设计稿基准宽度:  375px（适配320-430px）
水平边距:        16px（小屏）/ 20px（常规）
导航栏高度:      44px（含返回键）
底部标签栏:      56px + env(safe-area-inset-bottom)
底部安全区:      iOS: env(safe-area-inset-bottom) / Android: 0
```

## 二、入口流程 — 守护完整度检测（F-001）

### 页面 F-001-0：引导落地页

**目的：** 业务员分享链接的第一印象页，3秒内让陌生用户理解测评价值并开始作答，不感受销售压力。

**关键元素：**

- **业务员名片小条**（顶部，轻量设计）：圆形头像32px + 「XXX 为你发来一份礼物」，背景 #EBF4FF，圆角8px。不展示业务员电话和推销信息，语气轻松
- **主视觉区**（高度约240px）：温暖风格家庭守护插画（父母+孩子剪影，蓝绿渐变光环），主标题 `30秒，看看你的家庭守护有多完整`（24px / 700），副标题 `已有 128,432 个家庭完成了守护检测`（社会证明，13px / #6B7A8D）
- **价值点列表**（图标 + 文字，3项）：「生成专属五维守护报告」/ 「发现你的家庭守护盲区」/ 「生成可分享的守护健康证书」
- **CTA区**：主按钮全宽 `开始我的守护检测`，下方小字 `无需注册，30秒完成 · 完全免费`
- **底部信任背书**：`保障信息仅用于生成报告，不会被分享给第三方`（12px / #A8B4C0）

**用户流程：**
1. 通过业务员微信分享链接进入
2. 阅读主视觉和价值点（约3秒）
3. 点击「开始我的守护检测」→ Step 1

**状态设计：**
- 空状态：不存在（此页始终有内容）
- 加载状态：骨架屏，主视觉区灰色占位 + shimmer动效（#E8EDF3背景，1400ms循环）
- 错误状态：业务员信息加载失败时隐藏名片条，其余内容正常显示；若整页失败显示「页面加载失败，点击重试」

**关键文案：**
- 主标题：`30秒，看看你的家庭守护有多完整`
- CTA：`开始我的守护检测`
- 社会证明：`已有 128,432 个家庭完成了守护检测`
- 底部：`无需注册，30秒完成 · 完全免费`

## 附录 B：全局空状态设计原则

所有页面的空状态遵循以下5条原则：

1. **禁止显示空白**：所有空状态都有内容，告知用户「下一步怎么做」
2. **空状态也有参考值**：展示同龄人均值或示例数据（如「同龄人平均守护指数：65分」）
3. **引导文案正向**：不说「你还没有...」，说「完成XXX，就能看到...」
4. **插画选择**：使用暖色调温暖风格（非空洞几何图案）
5. **必须有CTA**：每个空状态至少有一个明确的操作入口

通用空状态文案模板：`你的[功能名]还在等待激活` + `完成[操作]，立即解锁`

| OCR拍照上传 | File API + FormData | `wx.chooseImage` + `wx.uploadFile` |
| 路由管理 | uni-app `uni.navigateTo` 统一处理 | uni-app `uni.navigateTo` 统一处理 |

> **uni-app 开发注意事项：**
> - Tailwind CSS 需配置 PostCSS，注意类名扫描范围（`content: ['./src/**/*.vue']`）
> - 微信H5和小程序的 `wx.share` API不同，用 uni-app 的 `uni.share` 统一处理
> - 积分「过期」使用软删除 + `expire_at` 字段，保留审计日志，不要硬删除
