## 全局交互规范

**加载状态（全局）：**
1. 页面首次加载：骨架屏（`van-skeleton`），不展示 Loading Spinner
2. 列表分页加载：底部 `van-list` 加载更多，文字「加载中...」
3. 按钮操作中：按钮变灰 + 文字变「处理中...」+ loading 图标，防止重复点击
4. 下拉刷新：`van-pull-refresh`，提示文字「下拉刷新」→「释放刷新」→「刷新成功」

**错误状态（全局）：**
1. 网络错误：全屏错误页，插图 + 「网络开小差了」+ 「重新加载」按钮
2. 局部错误：卡片内错误区域，小图标 + 简短说明 + 「重试」链接
3. 表单错误：字段下方红色提示文字，同时字段边框变红

**空状态（全局原则）：** 空状态必须包含「动作引导」，不能只显示「暂无数据」

### 页面 01：启动页

**目的：** 品牌认知，App 初始化加载期间的视觉过渡（≤ 2 秒），不拦截用户。

**关键元素：**
- 背景：渐变底色 `linear-gradient(160deg, #0D2B5E 0%, #1B6EF3 60%, #3B82F6 100%)`
- Logo 区域（垂直居中，上移 20%）：圆形品牌图标（48px）+ 品牌名（22px，白色，700）
- Tagline：「让每个家庭，活得更有底气」（14px，rgba(255,255,255,0.7)）
- 加载进度条：底部细线（高度 2px），从 0% 匀速走到 100%

**用户流程：**
1. App 启动 → 展示启动页，同步检测 Token 有效性
2. 加载完成（≤ 2s）→ 已登录跳今日工作台；未登录跳登录页
3. 超过 3s → 显示「正在努力加载...」

## 无障碍规范（WCAG 2.2 AA）

| 要求 | 实现方式 |
|------|---------|
| 色彩对比度 | 主文字 `#0F172A` on `#FFFFFF` = 19.6:1（超过 4.5:1 要求） |
| 触控目标 | 所有按钮最小 44×44px |
| 替代文字 | 所有图标按钮添加 aria-label |
| 焦点可见 | 所有可交互元素蓝色 2px outline focus ring |
| 减少动效 | 检测 prefers-reduced-motion，关闭粒子和缓动动效 |
| 颜色非唯一 | 热度信号同时使用颜色 + 文字标签双重区分 |

## 空状态设计原则（全局）

所有空状态必须包含四要素：
1. **插图**（线描风格，不超过 120×120px，SVG）
2. **主文案**（16px，`#0F172A`）：描述空状态原因
3. **副文案**（14px，`#64748B`）：激励性说明
4. **行动按钮**（蓝色主按钮）：引导用户执行下一步

|------|------|---------|

## 关键 UX 原则汇总

1. **打开即有事干**：首页工作台每次打开都有明确的「今日信号」，杜绝空白页面
2. **即时正反馈**：客户任何行为 < 3s 推达业务员；任务完成即时动效强化成就感
3. **一键操作**：发内容、记录跟进、拨打电话，任何操作路径 ≤ 3 步
4. **数据激励**：vs 昨日、vs 上周、排行榜、连续天数，多维度小赢设计
5. **新人保护**：入职 90 天内特殊引导逻辑，每日任务清单 + 高频正反馈
6. **渐进式信息**：卡片展示摘要，点击展开详情；列表页不过度呈现信息
7. **颜色即信号**：红/橙/绿/蓝热度色系贯穿全局，瞬间识别优先级
8. **不骚扰原则**：同一客户同一事件不重复推送；免打扰时段尊重
