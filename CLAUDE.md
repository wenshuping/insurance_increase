## 图标使用规范（强制）

**凡是涉及图标（Icon）的任何工作，必须使用 `better-icons` skill，通过 CLI 获取 SVG，禁止使用 emoji 或字符替代图标。**

```bash
# 搜索图标
npx better-icons search "icon-name"

# 获取 SVG 代码
npx better-icons get "lucide:icon-name"
```

**规则：**
- 禁止用 emoji 作图标（如 🏠 👥 📊 ✅）
- 禁止用 font-size 渲染字符图标
- 必须使用从 `better-icons` 获取的 inline SVG
- SVG 必须带 `stroke="currentColor"` 以支持颜色继承
- 推荐图标集：`lucide:*`（本项目首选）、`tabler:*`

## 文档输出规范

**严禁将任何文档直接创建在项目根目录。** 所有输出文件必须按以下规则存放：

| 文档类型 | 存放路径 | 示例文件名 |
|----------|----------|------------|
| 架构设计图、C4 图、系统设计 | `docs/architecture/` | `c4-投保流程.md` |
| API 接口文档 | `docs/api/` | `policy-api.md` |
| 数据库设计、ER 图 | `docs/database/` | `schema-v1.md` |
| PRD、需求文档、用户故事地图 | `docs/product/` | `prd-投保模块.md` |
| ADR 架构决策记录 | `docs/decisions/` | `ADR-001-技术选型.md` |
| 复盘报告、Bug 复盘 | `docs/postmortem/` | `postmortem-2026-02.md` |
| 设计系统、组件规范 | `design-system/` | `button-spec.md` |
| 测试报告、Playwright 报告 | `tests/reports/` | `test-report-投保流程.md` |
| 产品路线图 | `docs/` | `product-roadmap.md` |

**执行规则：**
- 创建文件前先确认目标目录存在，不存在则先创建目录
- 文件名使用中文或英文均可，但要清晰表达内容
- 同类文档按版本或日期区分，不覆盖历史记录
